Use the strategic-cto-mentor and systems-architect agents to help evaluate a technology decision using a structured framework.

Apply this decision framework:
1. **Frame** — identify all viable options clearly
2. **Clarify** — surface constraints (budget, timeline, team skills, compliance)
3. **Analyze** — for each option evaluate:
   - Total Cost of Ownership (3-year)
   - Time-to-value
   - Operational burden
   - Team fit and learning curve
   - Vendor lock-in risk
   - Risk profile
4. **Challenge** — aggressively question assumptions about costs and capacity
5. **Recommend** — clear path with evidence and rationale
6. **Define** — go/no-go criteria and what would change the recommendation

Common decision types this handles well:
- Build vs. Buy vs. Use open source
- Framework or language selection
- Database technology choice
- Cloud provider or multi-cloud strategy
- Monolith vs. microservices
- Vendor evaluation (tool A vs. tool B)

Output a clear recommendation with confidence level, key risks, and what information would change the decision.

If no decision is specified in the arguments, ask: "What technology decision would you like to evaluate?"
