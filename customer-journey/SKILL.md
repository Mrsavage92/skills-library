# Customer Journey Mapper

## Purpose

Produces a structured customer journey map: stages, touchpoints, customer actions, thoughts, emotions, pain points, and improvement opportunities. Outputs a map that teams can act on.

## When to Use

- Designing a new product or service experience from scratch
- Diagnosing where and why customers drop off or disengage
- Running a CX (customer experience) audit
- Aligning cross-functional teams (product, marketing, CS, support) on the customer view
- Preparing for a design sprint or service redesign

## Input Required

- Product/service being mapped
- Customer segment (be specific — "SMB founder using the product for X")
- Journey scope: pre-purchase only, post-purchase only, or end-to-end
- Any known pain points or data on drop-off

## Journey Stages

### Standard B2B SaaS Journey
1. **Unaware** — doesn't know the problem has a solution
2. **Problem Aware** — recognises the pain, starts researching
3. **Solution Aware** — discovers your category
4. **Brand Aware** — finds you specifically
5. **Evaluation** — trials, demos, compares alternatives
6. **Decision** — commits, signs up, pays
7. **Onboarding** — first 30 days, setup, first value
8. **Adoption** — regular use, team rollout
9. **Expansion** — upgrades, adds seats, new use cases
10. **Advocacy** — refers others, leaves reviews, case study

Adjust stages for your specific model (ecommerce, marketplace, services).

## For Each Stage, Map

| Element | Description |
|---------|-------------|
| **Touchpoints** | Where does the customer interact? (ads, website, email, sales call, product, support) |
| **Actions** | What does the customer do? |
| **Thoughts** | What are they thinking? ("Is this legit? Will this work for me?") |
| **Emotions** | Feeling: curious / confused / excited / frustrated / confident |
| **Pain points** | What slows them down or creates friction? |
| **Opportunities** | What could you do here to improve the experience? |
| **Owner** | Which team owns this stage? |

## Output Format

```

## Scope
[What's covered, what's excluded, key assumptions]

## Journey Overview
[One paragraph: the story of this customer's experience]

```
